# Feature Engineering

## Objective

The objective of this phase is to convert the cleaned textual job postings into numerical feature vectors that machine learning algorithms can understand and process.

---

## Dataset

- Input File: `processed_fake_job_postings.csv`
- Target Variable: `fraudulent`
- Feature Used: `clean_text`

---

## Train-Test Split

The dataset was divided into training and testing sets before feature extraction.

- Training Set: 80%
- Testing Set: 20%
- Random State: 42
- Stratified Sampling: Enabled

### Why Stratified Sampling?

The dataset is highly imbalanced:

- Genuine Jobs ≈ 95%
- Fraudulent Jobs ≈ 5%

Stratified sampling preserves the class distribution in both training and testing datasets, ensuring fair model evaluation.

---

## TF-IDF Vectorization

TF-IDF (Term Frequency–Inverse Document Frequency) was used to convert text into numerical vectors.

### Parameters Used

```python
TfidfVectorizer(
    max_features=10000,
    stop_words="english"
)
```

---

## Why TF-IDF?

TF-IDF was selected because:

- Captures the importance of words within documents.
- Reduces the influence of frequently occurring words.
- Performs well for text classification problems.
- Provides an effective baseline before experimenting with transformer-based embeddings.

---

## Feature Matrix

After vectorization:

- Training Shape: `(14304, 10000)`
- Testing Shape: `(3576, 10000)`

Each row represents one job posting.

Each column represents a vocabulary term generated by the TF-IDF vectorizer.

Successfully converted cleaned textual job postings into numerical feature vectors suitable for machine learning model training.
